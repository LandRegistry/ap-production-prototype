Getting started

Copy the Example (starter) flow.

  • Copy that into a new folder in views, and call the folder what you want
  • add a route to the main routes file (app/routes.js) - a route that looks at the folder you just created. see lines 12 to 19 to see what to copy. replace /example-flow with youer folder name
  • inside your views foler make the versions of your prototype, make folders names v1, v2 etc.
  • everything required to make a flow is contained in the example flow...
  • Use the notes within the code of the example flow to make your own flow

How the template (example) flow has been made

3 page layouts

  • documentation page - use {% extends "layouts/ap-documentation.html" %} at the top of your page
  • wide page, the two column, register and task page - use {% extends "layouts/ap-flow-wide.html" %} at the top of your page

There are 4 css files, refactoring is needed as there is duplication. Using the 3 layouts will load the correct css. If you need to add something, consider adding it to the 2 css files - regular and wide - for the two page width layouts we have

Scope of UI re-write

Technical UI re-write, into python/Flask, from react.

Limit the number of UI changes. This is about keeping as much as-is from the current UI, and the tech re-write brings the UI up to current HMLR code and design standards. It removes existing constraints that have limited user features, so new work post re-write can be done without working round the constraints.

UI changes are limited to those required by:

  • migrating to python/flask - some ui features may be difficult to migrate as-is from react. Where this is the case we'll redesign the ui feature to make it easy to implement
  • following common HMLR code and design standards - progressive enhancement, using gov components out of the box
  • fixing accessibilty failures in the current UI
  • minimising rework. Make some design changes - thats are considered to be foundational - for example responsive page layout - to avoid complex rework once all the UI features have been migrated
