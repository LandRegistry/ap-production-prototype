Combinations - Phase 3, DTC + RX1

Applications which contain any of the following transactions, combined into a full application flow:

  • Subsequent discharge
  • Transfer for value
  • Charge
  • Adding a restriction (RX1)

A combination in Application Processing will start with a summary page, which lists the inlcuded transactions and identifies where transactions may need to be completed in CORA. The first page of tasks in combinations is 'initial tasks' which cover the application level tasks. Transactions tasks are then presented in the order shown in the summary page.

Features of combinations include:

  • Ability to skip a transaction - If a user can't proceed with a certain transaction they can skip it, to then continue processing in AP. The 'cancel processing this transaction' link replaces the 'return to workflow' link and takes the user to an interim summary page where they can choose to either continue processing or drop the whole case out to Workflow.
  • Statuses - If a transaction has been skipped it will appear as 'Not drafted' in subsequent summary pages. The status of the transactions is confirmed in the final summary screen with information about what happens next and a dynamic button for if the case will submit to markoff or be sent to Workflow for the resolution of PAS notes.
  • Partial saving - Transactions are saved once a user has moved past them. The 'save and continue' button at the end of each transaction marks each drafting save point.
