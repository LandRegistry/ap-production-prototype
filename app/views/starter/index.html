Starter flow

A basic flow you can build actual flows from, containing:

  • worklist (to open a case from)
  • preliminary page (initial questions and verified by automation)
  • two 'middle' flow page
  • document retention
  • confirmation pages (the 3 types)
  • worklist (after completing the case)
  • document viewer
  • accessibility and feedback pages
  • prototype tools page (to clear data, and see saved session data)

The starter folder has a self contained _routes.js file, containing everything needed operate the flow. But, when copying this starter folder to create your own flow, you will need to add a route to the main routes file (at the top level of the protoype kit, app/routes.js), to tell the maim routes file to look for your flow routes in your folder.
