Design history notes

29 November 2023:

Moe has provided content updates/amends - these are implimented in these updated designs.

20 November 2023:

We need to understand what kind of validation will be applied to the main misc info textarea (input field); is there any form of parsing/filtering to stop case workers inputting any manner of inappropriate information?

15 October 2023:

Option 1.3: This is the preferred option. It integrates smoothly with the established flow. Its core Yes or No option/question can be moved easily to accommodate future changes to the flow as and when additional functionality is added (with increasing data input options being ported from CORA). (Lots of user research and resource already invested in the current flow for option 2 – significantly different method of approaching the functionality – to be invalid).

For this piece of design work we attempted to recreate how the 'live' service currently appears (for a smoother transfer of new form designs, ensuring that they work in the actual enviroment and can be followed accurately. 'Live' is currently in transistion to the 'new' AP design aesthetic). There are small discrepancies with live and these design pages.

An additional 'hint' text statement added to the first page to encourage users to add as many transferees as possible at the start of this combined journey.

Where the user is served confirmation that the misc info has been added the page reminds the user which misc info (and related names) has been added. (These are added to the bottom of any list – to keep the numbering consistent). This design is a derivation on the new GDS summary cards … these summary cards are compromised when used on our grey page background, so this subtle adjustment is a way to resolve this. Using this list is a convenient way of resolving width issues in the limited screen real-estate when resizing the window. (This design is something I have used and tested previously, and a standalone matching version could also be created as a wrapper for chunking info). This to be discussed with the wider UCD (with the current table design – as used for addresses – fallback option).

Form validation prototype pages are work in progress. Content design revisions – for language – to follow. New flow to be captured in Figma.

Notes: This is not a fully functional prototype (yet), and is a series of clickable design prototypes/assets - without form validation and with some links/buttons disabled. They use HTML recently supplied from the live/active service (and which have been updated where appropriate to reflect the latest designs (opens in a new tab) as shown in Figma - 'Final designs: enter UK postal addresses').
