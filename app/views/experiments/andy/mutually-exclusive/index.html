when the caseworker is asked for a or, b

current examples are:

  • charge amount (input it), or not stated (say not stated)
  • charge date (input it), or no date (say no date)
  • document retention - whats the date of this document, or not given/cant read it

Gov design system has mutually exclusive pattern for checkboxes

We want to have an input field(s) or a check box

We dont want to ask a yes/ no question first - as almost all the time the caseworker will have the charge amount or date so we want them to just enter that and say not present occasionally. Optimise around its always present

ONS have a mutually exclusive component - similar in behaviour to the Gov design system, for more than just checkbox list - that looks just like the circumstances we want: Date | Number

we need a non Javascript example. and can progressively enhance it for Javascript users (nearly all). We like the ONS behaviour, but make it GOV styled - e.g. no borders round the checkbox, and GOV error styling.

In the current way we do charge date and amount, we have Javascript behaviour that deactivates the date input. We want to avoid deactivating, and improve the pattern making it more GOV like.
