Gov design system has a mutually exclusive pattern for checkboxes

We want to have an input field(s) or, a check box

In the current way we do charge date and amount, we have Javascript behaviour that deactivates the date input. We want to avoid de-activating*, and improve the pattern visually making it more GOV like.

We dont want to ask a yes/ no question first - as almost all the time the caseworker will have the charge amount or date so we want them to just enter that and say, not present only occasionally. Optimise around - it is always present

ONS have a mutually exclusive component* - similar in behaviour to the Gov design system checkbox, for more than just checkbox list - that looks just like the circumstances we want: Date | Number

*note: porting over the ONS js code and html, but not the css illustrates the assistive technogy features built in to the ONS pattern (to make it pass WCAG accessibilty requiremements - presume these do make it pass). If we continue with de-activating, or other behaviour, we also need to make it accessible. Don't know what assistive technology feature code is in production currently.

We need a non Javascript version. and can progressively enhance it for Javascript users (nearly all). We like the ONS behaviour, but lets make it GOV styled - e.g. no borders round the checkbox, and GOV error styling.

We need to document how the non js works, and write error messages for the non js version. ONS and Gov checkboxes have the non js error messages behaviour we want to copy.
