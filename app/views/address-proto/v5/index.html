Custom lender entry

Which design is represented?

Design for creating a custom lender entry as part of the Charge flow. User is asked to enter the name of the lender(s) and their addresses. The design shows the future scenario in which UK postal addresses and email addresses are accepted .

  2. Email address: arthurboy91@mail.com. Address only for Arthur Boyle.

What are the steps that the user is asked to follow in the prototype?

  1. Users starts from the MD ref / register entries page.
  2. In order to start the sub-flow for creating a custom lender entry, they have to press the 'Create custom lender entry' button within the register entries section.
  3. By doing so, users enter a sub-flow.
  4. The first step of the sub-flow asks them to enter the names of all the lenders. In this prototypes there are two names to be added.
  5. Once all names have been added to the page, users presses 'next'. Doing so brings them to a page showing the empty addresses table. The page is still withing the subflow.
  6. Users need to enter the details of the addresses before they can press 'create custom lender entry'. They do so by pressing on 'add address'.
  7. They'll then select the type of address they want to add and press next. In the prototype the first address they have to select is UK postal address.
  8. User completes the required details about the address and press 'save address'.
  9. The UK postal address gets so added to the table.
  10. Users can now enter the details of the second address. They press on 'add address' and enter the email address in the subsequent step, before saving 'save address'
    11. The second address is then added to the table. Now users can press 'create custom lender entry', having also answered 'Other than UK postal addresses is there anything else you want to add? '
  11. The newly created custom lender entry gets so added to the 'register entries' section for them to double check and eventually amend/remove.
  12. End of the flow. They can press 'next' or 'return to workflow' to go back to the index.
