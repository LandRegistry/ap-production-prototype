Address for service

Which design is represented?

Designs in which the accepted addresses are only UK postal address (so the user will not be asked to select which type of address they want to add - as only one type of address is accepted)

What are the transferee names in the prototype?

Olivia West and Oscar Williams

Addresses to be added in the prototype

  1. UK postal address: 15 Pevensey Rd, Eastbourne, BN21 3HR. Address for both transferees.

  2. UK postal address: 127-129 Manchester Rd, Rossendale, Lancashire, BB4 5SL. Address only for Olivia West.

What are the steps that the user is asked to follow in the prototype?

  1. Users starts from the transferees page. The transferee names are pre-populated by the system.
  2. They then get into a page showing an empty address table. They are so required to start populating the table by pressing on 'add address'.
  3. They'll then fill the details for address number 1 (UK postal address), and select which transferees the address belongs to by ticking the checkbox with the transferee names before pressing 'save address'.
  4. Address number 1 will so get added to the table.
  5. Users now have to enter address number 2. They do so by pressing on 'add address'.
  6. They'll then fill the details for address number 2 (which also is a UK postal address), and select which transferees the address belongs to by ticking the checkbox with the transferee names before pressing 'save address'.
  7. Address number 2 will so get added to the table.
  8. At this point the number of addresses that can be added in the prototype is reached , and the user can press next to move to the next page of the Transfer flow, having answered the 'Other than UK postal addresses is there anything else you want to add? ' question.
