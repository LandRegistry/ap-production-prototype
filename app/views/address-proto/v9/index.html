Design for creating a custom lender entry as part of the Charge flow. User is asked to enter the name of the lender(s) and their addresses. The design shows the scenario in which only UK postal addresses are accepted.

Which design for when the address type is not accepted in AP is shown?

Design in which the user goes through a 'filter page' after entering the lender details. Communicating on the 'filter page' to the system that the addresses the user wants to add are NOT ONLY UK POSTAL ADDRESSES will cause the rejection of the application. '

What are the steps that the user is asked to follow in the prototype?

  1. Users starts from the MD ref / register entries page.
  2. In order to start the sub-flow for creating a custom lender entry, they have to press the 'Create custom lender entry' button within the register entries section.
  3. By doing so, users enter a sub-flow.
  4. The first step of the sub-flow asks them to enter the names of all the lenders. In this prototypes there are two names to be added.
  5. Once all names have been added to the page, users presses 'next'.
  6. User then gets to a 'filter page'. If they only have to enter UK postal addresses, the system will let them move on. Otherwise the application gets rejected.
  7. Users need to enter the details of the addresses before they can press 'create custom lender entry'. They do so by pressing on 'add address'.
  8. They are then shown the address lookup design (as the only type of address accepted in this phase are UK postal addresses).
  9. User completes the required details about the address and press 'save address'.
  10. The address gets so added to the table. Now users can press 'create custom lender entry'. '
  11. The newly created custom lender entry gets so added to the 'register entries' section for them to double check and eventually amend/remove.
  12. End of the flow. They can press 'next' or 'return to workflow' to go back to the index.
